Using multiple domains?

You can skip the “login per domain” dance by using EXT:multisite_belogin (cross site/domain backend login).

It’s especially handy when editors use backend tools in the frontend (preview / editing overlays) across domains.

Why am I seeing this?
Backend login is stored in browser cookies. If you’re logged in on a different domain (or a different protocol like HTTP vs HTTPS), this domain may not have the cookie it needs—so the visual editor can’t verify your session.
Troubleshooting tips
• Clear browser cookies/cache for this site and try again (old cookies can block a fresh login).
• Try another browser or a private window to rule out cookie conflicts.
• Ensure the backend login is done via HTTPS on the correct domain.
If you still bounce back to the login page or sessions don’t stick, the issue is often cookie configuration or caching on the server side.
Admin notes for EXT:multisite_belogin
The extension is intended for cross site/domain backend login. Typical requirements include setting cookie SameSite appropriately (often cookieSameSite = none), using full qualified domain names (FQDN) for site bases, and logging in via HTTPS.
Not your session? Use another browser profile, or log out in the backend and log in again.
